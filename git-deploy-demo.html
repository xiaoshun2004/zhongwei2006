Git自动部署演示

体验现代化的网站更新流程

📊 部署方式对比

手动上传方式

  • 1. 本地修改文件
  • 2. 压缩整个项目为ZIP
  • 3. 登录Netlify控制台
  • 4. 删除旧部署
  • 5. 上传新的ZIP文件
  • 6. 等待部署完成
  • 7. 测试网站是否正常
⏱️ 平均用时: 5-10分钟

Git自动部署（推荐）

  • 1. 本地修改文件
  • 2. 提交到Git仓库
  • 3. 推送到GitHub
  • 4. 自动触发部署
  • 5. 网站自动更新
git add .
git commit -m "更新内容"
git push
⚡ 平均用时: 30秒-2分钟

🎯 实战演示

模拟更新场景：修改首页标题

需求：将首页标题从"优质企业"改为"LED导光板专家"

# 1. 修改 index.html 文件
<title>LED导光板专家 - 18年专业技术</title>

# 2. 提交更改
git add index.html
git commit -m "更新：修改首页标题"
git push
预计30秒后网站自动更新完成！

📋 设置Git自动部署步骤

  1. 删除当前网站：在Netlify控制台删除手动上传的网站
  2. 连接GitHub：创建新网站，选择"New site from Git"
  3. 选择仓库：选择您的GitHub仓库 xiaoshun2004/zhongwei2006
  4. 配置设置：发布目录留空，构建命令留空
  5. 完成部署：等待首次自动部署完成
